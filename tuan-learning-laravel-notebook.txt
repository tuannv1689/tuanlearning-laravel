Laravel - Configuration:
    - read the env: 'env' => env('APP_ENV', 'production'),
    - enable caching: php artisan config:cache
    - maintenance mode: php artisan down, php artisan up
Laravel - Routing:
    - routes are registered within the app/routes.php, or app/Http/routes.php
    - route return an blade: return view('welcome');
    - example: 
        app/Http/routes.php:
            <?php
            Route::get('/', function () {
               return view('welcome');
            });
        resources/view/welcome.blade.php: html code
    - Route Parameters: Route::get('ID/{id}',function($id) {echo 'ID: '.$id;});
    - Optional Parameters: Route::get('user/{name?}', function ($name = 'TutorialsPoint') { return $name;});
    - Named Routes: Route::get('user/profile', 'UserController@showProfile')->name('profile');
                    The user controller will call for the function showProfile with parameter as profile. 
                    The parameters use name method onto the route definition.
Laravel - Middleware: Middleware acts as a bridge between a request and a response
    - create: php artisan make:middleware <middleware-name>
    - can be seen at app/Http/Middleware directory.
    - The Global Middleware will run on every HTTP request of the application, 
    - whereas the Route Middleware will be assigned to a specific route
    - The middleware can be registered at app/Http/Kernel.php, This file contains two properties $middleware and $routeMiddleware
    - Add routeMiddleware to a controller: app/Http/routes.php: 'middleware' => 'Role:editor',
    - Route middleware is call before the controller
    - function terminate() is called after the controller
Laravel - Namespaces:
    - The default namespace used in Laravel is App
    - Create namespace SocialNet: php artisan app:name SocialNet
Laravel - Controllers
    - Create: php artisan make:controller <controller-name> # add --plain to ignore CRUD
    - Route by: Route::get(‘base URI’,’controller@method’);
    - Assigning Middleware to Route
        Route::get('profile', [
           'middleware' => 'auth',
           'uses' => 'UserController@showProfile'
        ]);
    - Assigning Middleware within Controller’s constructor
        $this->middleware('auth');
    - CRUD (Create, Read, Update, Delete)
    - Route the CRUD: Route::resource('my','MyController');
        Verb	    Path	        Action	Route Name
        GET	        /my	            index	my.index
        GET	        /my/create	    create	my.create
        POST	    /my	            store	my.store
        GET	        /my/{my}	    show	my.show
        GET	        /my/{my}/edit	edit	my.edit
        PUT/PATCH	/my/{my}	    update	my.update
        DELETE	    /my/{my}	    destroy	my.destroy
    - Implicit Controllers: /test/show/1 -> getShow($id) , /test/admin-profile -> getAdminProfile()
        POST /test/profile -> postProfile()
        then: Route::controller('test','ImplicitController');
    - Constructor Injection: 
        app/Http/routes.php
            class MyClass{
               public $foo = 'bar';
            }
            Route::get('/myclass','ImplicitController@index');
            
        app/Http/Controllers/ImplicitController.php:
             public function __construct(\MyClass $myclass) {
                 dd($this->myclass); # dd = dump a class
             }
    - Method Injection: 
                 dd($myclass);
Laravel - Request:

        $request->path() = requested URI (no website url, only params)
        $request->is() = true/false match result
        $request->url () = full url
        $request->input ('username') = username value in url, or can use: $request->username
    - csrf_token: a secure random token used to prevent CSRF attacks.

Laravel - Cookie:  
    Cookie can be created by global cookie helper of Laravel. It is an instance of Symfony\Component\HttpFoundation\Cookie
    
    The cookie can be attached to the response using the withCookie() method. 
    
    Create a response instance of Illuminate\Http\Response class to call the withCookie() method
    
    Cookie generated by the Laravel are encrypted and signed and 
    it can’t be modified or read by the client.
    
    - set cookie:
        //Create a response instance
        $response = new Illuminate\Http\Response('Hello World');
        //Call the withCookie() method with the response method
        $response->withCookie(cookie('name', 'value', $minutes));
        //return the response
        return $response;
    - get cookie: $value = $request->cookie('name');
